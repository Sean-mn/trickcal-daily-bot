# 트릭컬 노트 API/데이터 구조 파악 결과
실제 API 엔드포인트:

https://api.note.trickcal.com/note/data — 유저 저장 데이터 불러오기 (쿠키 인증 필요)
https://api.note.trickcal.com/note/signout — 로그아웃 (POST)
https://api.note.trickcal.com/note/revoke-all — 전체 세션 해제 (POST)

## 중요한 점
게임 데이터(사도 정보, 보드, 장비, 연구 등)는 JS 번들에 내장되어 있고, /note/data API는 개인 유저 저장 데이터(보유 사도, 보드 진행 상황 등)를 주고받는 구조입니다. 이 API는 쿠키 기반 인증을 사용하므로, 외부에서 공개 API처럼 자유롭게 호출하기는 어렵습니다.

디스코드 봇에 활용 가능한 기능 아이디어
:mag: 웹 스크래핑 기반 (인증 불필요)
트릭컬 노트 사이트를 직접 파싱해서 구현할 수 있는 기능들입니다.
사도 정보 검색 — /personal 페이지에 전체 사도 목록이 있고, CDN 이미지 URL 패턴(cdn.note.trickcal.com/HeroIcons/[id].webp)도 확인했습니다. 사도 이름, 성격(타입), 종족, 초기 성급 등의 정보를 제공하는 /사도 루포 같은 명령어를 만들 수 있어요.
보드 스탯 계산기 — /boardsearch 페이지를 보면 HP, 물리 공격력, 마법 공격력, 물리 방어력, 마법 방어력, 치명타, 치명 피해, 치명타 저항, 치명 피해 저항 등 9가지 스탯 우선순위를 설정하고 보드 조합을 추천받는 기능이 있습니다. 디스코드에서 스탯 조건을 입력하면 추천 사도/보드 조합을 알려주는 기능으로 구현 가능합니다.
연구소 재료 계산 — /lab 페이지의 데이터를 분석해보니 연구 단계별 필요 재료(골드, 시간, 아이템)가 번들에 내장되어 있어요. 현재 연구 단계를 입력하면 남은 재료를 계산해주는 기능을 만들 수 있어요.
장비 검색 — /equipsearch에서 사도별로 어떤 장비가 적합한지 검색 기능이 있습니다. 디스코드에서 사도명으로 장비 추천을 받는 명령어를 구현할 수 있어요.


## 추천 구현 우선순위
인증 없이 바로 구현 가능하고 유용한 것들부터 하면 이렇습니다.

사도 정보 카드 — 사도 이름 검색 시 성격, 종족, 성급, 이미지를 임베드로 표시
보드 스탯 추천 — 원하는 스탯 조합 입력 → 추천 사도/보드 조합 출력
연구소 재료 계산기 — 현재 단계 ~ 목표 단계까지 필요한 재료 계산
장비 조합 추천 — 특정 사도의 추천 장비 세트 안내
